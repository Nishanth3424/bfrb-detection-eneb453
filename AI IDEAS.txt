# High-Precision BFRB Detection from MediaPipe Landmarks

## Executive Overview

This report proposes concrete, point-based rules for detecting body-focused repetitive behaviors (BFRBs) from MediaPipe Holistic hand and face landmarks with a strong bias toward **zero or near-zero false positives**, even if that lowers recall.
The design builds directly on an existing rule-based engine that already uses face-width normalization, proximity, velocity, dwell time, pinch strength, fingertip clustering, and pinch–pull distance to detect BFRBs such as hair pulling, skin picking, nail biting, nose picking, lip picking, and general face touching.[1]

Key ideas from prior work on BFRB wearables, eating detection, and gesture systems are adapted to a webcam + landmark setting: use **multi-stage gating**, **temporal windows**, and **context-specific negative detectors** (especially for eating and neutral face-resting) rather than relying on a single static condition like “all fingertips are close together.”[2][3][4]

The result is a set of formulas and heuristics organized into:

- Global gating signals that every BFRB must satisfy
- Negative detectors to explicitly block eating and passive face-resting
- Per-BFRB rules (hair pulling, skin picking, nail biting, nose picking, lip picking, generic face touching) expressed purely in terms of landmark distances, angles, and temporal behavior


## 1. Background and Constraints

### 1.1 Existing engine summary

The current system already uses MediaPipe Holistic to get 21 hand and 468 face landmarks per frame, then computes distances in normalized image space.[1]
Key components that are already implemented include:

- **Face width** \(faceWidth = dist(face, face)\) to normalize all distances into “face-width units,” making thresholds scale-invariant.
- **Proximity**: minimum normalized distance between key hand landmarks (wrist, index tip, thumb tip, grab point) and key face landmarks (nose, cheeks, lips, forehead).[1]
- **EMA smoothing** of distances, velocities, pinch strength, and cluster strength to suppress frame-to-frame jitter.
- **Velocity filter** based on wrist motion (slow movements only are eligible for BFRBs).
- **Dwell timer**: hand must remain near the face for at least \(500–1000\) ms, with low velocity, before a BFRB is declared.
- **Pinch strength** from thumb–index and thumb–middle distances, normalized by face width.
- **Fingertip cluster strength** from the average spread of all five fingertips around their centroid.
- **Pinch–zone proximity**: grab-point near specific face zones (forehead, cheeks, chin, nose).
- **Pull-back distance**: difference between current hand–face distance and the distance at pinch start to confirm hair-pulling.

These ingredients are strong, but a single condition like “all fingertips are close together” still fires during **eating** (grasping utensils, holding food) and **neutral face-resting** (leaning the head on a hand), because those behaviors can also produce clustered fingertips or a pinch-like pose near the face.


### 1.2 Lessons from related work

Research on BFRB detection with **wearables** shows that high accuracy comes from combining multiple sensing channels (motion, orientation, heart rate, thermal) and using temporal windows of several seconds, not just single-frame signals.[5][3][4]
Similarly, eating-detection systems find that delaying confirmation until several gestures are observed greatly reduces false positives from unrelated hand-to-mouth movements.[2]
Gesture-recognition work also emphasizes training on “noise classes” (irrelevant but frequent gestures) and applying personalization/calibration to reduce false activations in real-world use.[6]

For face-centric gestures, hand-over-face detectors treat **occlusion of the face** as a primitive that other detectors can use to veto or confirm events, illustrating the value of explicit “gating” components.[7]

Translating these ideas to a landmark-only webcam system suggests three design principles:

1. Use **multi-stage gating**: a gesture should pass several independent tests (location, shape, temporal pattern, and context) before being labeled as BFRB.
2. Use **temporal windows** and micro-motion patterns: BFRBs are characterized by repetitive micro-movements at a fixed location, whereas eating or neutral resting have different temporal signatures.
3. Add **negative detectors** for common confounds like eating and passive face-resting so that a BFRB is only allowed when no negative detector is active.


## 2. Global Landmark-Based Signals

This section defines reusable geometric and temporal signals derived from hand and face landmarks.
All distances are normalized by \(faceWidth\) as in the existing system.[1]

### 2.1 Basic distances

Use the existing Euclidean distance in 2D:

\[
\text{dist}(A, B) = \sqrt{(A_x - B_x)^2 + (A_y - B_y)^2} \quad (1)
\]

Normalize any distance \(d\) by

\[
\tilde d = \frac{d}{faceWidth} \quad (2)
\]

Key points:

- Hand: wrist (0), thumb tip (4), index tip (8), middle tip (12), ring tip (16), pinky tip (20)
- Face: nose tip (1), forehead center (10), upper lip (13), lower lip (14), chin (152), left cheek (234), right cheek (454)[8][1]


### 2.2 Proximity and zones

Reuse and slightly refine the existing minimum proximity distance:

- \(d_{w,n} = \tilde{\text{dist}}(wrist, nose)\)
- \(d_{i,u} = \tilde{\text{dist}}(indexTip, upperLip)\)
- \(d_{t,u} = \tilde{\text{dist}}(thumbTip, upperLip)\)
- \(d_{i,n} = \tilde{\text{dist}}(indexTip, nose)\)
- \(d_{i,c_L} = \tilde{\text{dist}}(indexTip, leftCheek)\)
- \(d_{i,c_R} = \tilde{\text{dist}}(indexTip, rightCheek)\)
- \(d_{g,f} = \tilde{\text{dist}}(grabPoint, forehead)\)

Then define

\[
\tilde d_{min} = \min(d_{w,n}, d_{i,u}, d_{t,u}, d_{i,n}, d_{i,c_L}, d_{i,c_R}, d_{g,f}) \quad (3)
\]

Smooth \(\tilde d_{min}\) with EMA as already implemented.[1]
A hand is considered **near the face** if

\[
\tilde d_{min}^{(sm)} < 0.35 \quad (4)
\]

The existing **zones** (forehead, cheeks, chin, nose) defined by a center landmark and a radius in face-width units remain valid and should be reused.[1]


### 2.3 Mouth opening and chewing signal

To differentiate BFRBs from eating, exploit **mouth opening** and its temporal pattern.
Define

\[
M_t = \tilde{\text{dist}}(face[9], face[10]) = \frac{\text{dist}(\text{upperLip}, \text{lowerLip})}{faceWidth} \quad (5)
\]

Interpretation:

- \(M_t < M_{closed}\): mouth essentially closed (e.g., \(M_{closed} \approx 0.03\)).
- \(M_t > M_{open}\): mouth clearly open for eating or talking (e.g., \(M_{open} \approx 0.07\)).

Track \(M_t\) over a sliding window of \(T_M\) frames (e.g., \(T_M \approx 60\) at 30 fps ≈ 2 s) and compute:

- Mean opening \(\bar M\)
- Standard deviation \(\sigma_M\)
- Number of peaks where \(M_t\) crosses from below \(M_{open}\) to above it (chewing/bites count)

Eating tends to produce repeated large openings (high \(\sigma_M\), multiple peaks), whereas nail biting, lip picking, or skin picking usually keep the mouth nearly closed or only slightly open.[2]


### 2.4 Wrist and fingertip micro-motion

The current engine uses smoothed wrist velocity to filter out fast motions.[1]
To further split BFRBs from neutral face-resting, track **micro-motions** in a short window.

For any scalar signal \(x_t\) over a window of \(T\) frames (e.g., 30–60 frames):

- Amplitude \(A_x = \max(x_t) - \min(x_t)\)
- Zero-crossing count of \(\Delta x_t = x_t - x_{t-1}\) (how often it changes direction)

Useful choices of \(x_t\):

- Distance from grabPoint to a zone center (for hair pulling and skin picking)
- Mouth opening \(M_t\) (for chewing vs still mouth)
- Thumb–index distance (for pinch pulsations)

Heuristically:

- BFRBs show **moderate amplitude** micro-motions (e.g., \(0.01–0.05\) face widths) and **frequent direction changes** (scratching, picking, nibbling at nails).
- Neutral face-resting shows **near-zero amplitude** and very few direction changes (the hand is static).


### 2.5 Pinch and cluster refinement

The system already defines **pinch strength** (thumb–index and thumb–middle) and **fingertip cluster strength**.[1]
To reduce false positives from generic pinching (e.g., holding objects while eating), add a simple **shape descriptor**:

Let

- \(v_{TI} = hand[11] - hand[5]\) (index tip minus thumb tip, the pinch axis)
- \(v_{WI} = hand[11] - hand\) (index tip minus wrist, finger direction)

Define the cosine of the angle between these vectors:

\[
\cos \theta_{pinch} = \frac{v_{TI} \cdot v_{WI}}{\lVert v_{TI} \rVert\, \lVert v_{WI} \rVert} \quad (6)
\]

Intuition:

- For **nail biting** or **lip picking**, the index finger tends to point roughly into the mouth, and thumb–index are aligned in that direction, so \(\cos \theta_{pinch}\) is relatively high (vectors roughly colinear).
- For **holding an object** sideways (utensil, fork), the pinch axis is often closer to perpendicular to the direction from wrist to index tip, yielding lower \(\cos \theta_{pinch}\).

This is not perfect, but it adds an extra discriminator that is hard to satisfy accidentally when eating if thresholds are tuned conservatively.


## 3. Global Gating and Negative Detectors

Every BFRB detection should be allowed only if it passes a **global gaiting pipeline** and no negative detector fires.

### 3.1 Global positive gates (must all be true)

For any BFRB event candidate:

1. **Near-face gate**: \(\tilde d_{min}^{(sm)} < 0.35\) for at least some small number of frames (e.g., 5–10).
2. **Slow-motion gate**: smoothed wrist velocity \(v_w^{(sm)} < 0.08\) in face-width units per frame (current threshold).
3. **Dwell gate**: total dwell time in the zone ≥ \(T_{dwell}^{min}\) (e.g., 800–1200 ms), except for hair-pulling pinch–pull, which can trigger faster due to the unambiguous pull-back.[1]
4. **Hand–face occlusion gate** (optional): a crude check that the projected hand bounding box overlaps the face bounding box to ensure true contact rather than wide passing.


### 3.2 Negative detector: eating / drinking

Use a combination of hand-to-mouth proximity, mouth opening, and temporal pattern to explicitly recognize eating-like episodes and veto BFRB events.

Define an **eating score** over a sliding window:

- Hand near mouth: \(\tilde{\text{dist}}(indexTip, upperLip) < 0.25\) or \(\tilde{\text{dist}}(thumbTip, upperLip) < 0.25\) for at least \(T_{nearMouth}\) ms.
- Mouth opening: \(M_t > M_{open}\) frequently (e.g., mouth open for ≥ 30–40% of frames in the window).
- Chewing/bite pattern: at least 1–2 large peaks in \(M_t\) where \(M_t\) crosses above \(M_{open}\) and back down.

If these three conditions hold in the last \(T_E\) seconds (e.g., 2–4 s), declare **eatingLikely = true**, and **suppress all BFRB detections** in that window.
This follows the same idea as When2Trigger, which trades a small detection delay for much lower false positives in eating detection.[2]


### 3.3 Negative detector: passive face-resting

A second negative detector aims at the case where the person simply rests their head on their hand.

Signals:

- Hand near cheek or jaw zone (cheek or chin zone active).
- Wrist and fingertip micro-motion amplitudes **below** a very low threshold (e.g., amplitude of grabPoint distance to zone center \(< 0.01\) face widths over 1–2 s and very few direction changes).
- Fingertip cluster strength low or moderate (hand is more flat/palm-based than tightly clustered).

If these conditions hold for \(T_R\) seconds (e.g., > 2 s), declare **restingLikely = true** and veto BFRB detections in that interval.


## 4. High-Precision Rules per BFRB Type

The following sections assume that the global positive gates are satisfied and that **eatingLikely == false** and **restingLikely == false**.
Thresholds are expressed in face-width units and should be tuned empirically with live data.


### 4.1 Hair pulling (scalp, eyebrow, eyelash)

Hair pulling is characterized by a **pinch near the scalp or brow** followed by a **pull-back movement away from the head**.[1]

#### 4.1.1 Detection conditions

**Shape and location:**

- Strong pinch: smoothed pinch strength \(S_{pinch}^{(sm)} > 0.6\).
- Fingertip cluster: \(S_{cluster}^{(sm)} > 0.2\) (fingers are somewhat clustered, but not fully bunched like skin picking).
- Grab point in hair zone: \(\tilde{\text{dist}}(grabPoint, forehead) < R_{hair}\) with \(R_{hair} \approx 0.8\) as already used for scalp.[1]

**Temporal pattern:**

1. At time \(t_0\), pinch becomes strong and zone condition becomes true (pinch-onset).
2. Record \(\tilde d_{hair}(t_0) = \tilde{\text{dist}}(grabPoint, forehead)\).
3. Within \(T_{pull}\) ms (e.g., 500–800 ms), require a pull-back:
\[
\tilde d_{hair}(t) - \tilde d_{hair}(t_0) \geq 0.28 \quad \text{for some } t \in [t_0, t_0+T_{pull}] \quad (7)
\]

**Micro-motion filter:**

- During the pull-back, wrist velocity should remain moderate (not a sudden large swing), and the pull distance should be monotonic increasing or mostly so.
- After the pull, the pinch may relax; that is acceptable.

**Additional discriminators vs scratching or adjusting hair:**

- **Minimal pre-pinch micro-motion**: in the \(0.5\) s before \(t_0\), require relatively low micro-motion near the zone so that a long scratch (back-and-forth) does not qualify.
- **Single distinct pull cluster**: multiple small oscillations in \(\tilde d_{hair}\) without a clear net increase above \(0.28\) face widths should not trigger.

This rule is already close to the existing pinch–pull engine; the main additions are the explicit micro-motion filter and the requirement that the pull be a **net outward movement** rather than oscillatory scratching.[1]


### 4.2 Skin picking (cheek, forehead, chin, neck)

Skin picking involves **multi-finger clusters** that remain on a small skin patch and perform tiny repetitive motions, often without a large pull-away.[1]

#### 4.2.1 Detection conditions

**Shape and location:**

- Strong cluster: smoothed cluster strength \(S_{cluster}^{(sm)} > 0.4\) (all fingertips close together).
- Pinch may or may not be strong; require **either** pinch strength \(> 0.4\) or high cluster strength.
- Grab point or fingertip centroid inside a skin zone (cheeks, chin, forehead, neck), e.g.:
  - Left cheek: \(\tilde{\text{dist}}(centroid, leftCheek) < 0.55\)
  - Right cheek, chin, neck similarly (existing radii).[1]

**Temporal pattern:**

Over a window \(T_P\) (e.g., 1.5–2 s) while near the zone:

- Dwell time ≥ \(T_P\) with \(v_w^{(sm)} < 0.08\).
- Micro-motion amplitude of grabPoint distance to zone center \(A_g\) in \(0.01–0.05\) face-widths.
- Number of direction changes in \(\tilde{\text{dist}}(grabPoint, zoneCenter)\) above a small threshold (e.g., ≥ 3–4 zero crossings), indicating repeated small picking motions.

**Resting veto:**

- If \(A_g < 0.01\) and direction changes ≈ 0–1, treat as restingLikely and veto.

This discriminates between **static resting** (very low micro-motion) and **active skin picking** (small, repeated adjustments of fingertip position on the skin).


### 4.3 Nail biting

Nail biting is a **hand-to-mouth dwell** where 1–2 fingertips stay at the upper lip/teeth region, with small oscillatory motion of the finger but without the large mouth openings and hand trajectories characteristic of eating.[1]

#### 4.3.1 Detection conditions

**Shape and location:**

- Active fingertip: either indexTip or thumbTip.
- Proximity: \(\tilde{\text{dist}}(indexTip, upperLip) < 0.18\) or \(\tilde{\text{dist}}(thumbTip, upperLip) < 0.18\).
- Other fingertips not clustered: cluster strength moderate or low (nail biting typically uses one finger, not a full cluster).
- Pinch shape: \(S_{pinch}^{(sm)}\) between \(0.2\) and \(0.6\) and \(\cos \theta_{pinch}\) above a threshold (e.g., > 0.4), indicating thumb–index alignment into the mouth rather than sideways object holding.

**Temporal pattern:**

- Dwell time near mouth ≥ \(T_N\) (e.g., 1.5–2 s).
- Mouth opening small: \(M_t < M_{open}\) for most of the window (no large chewing-like openings).
- Micro-motion: fingertip distance to upper lip shows small oscillations (amplitude \(0.01–0.03\) face widths) with several direction changes.

**Eating veto:**

- If mouth opening shows large peaks or repeated wide openings (chewing pattern) in the same window, treat as eatingLikely and veto.

This distinguishes **nail biting** from **eating**, where mouth opening and hand trajectory are larger, and multiple fingers or utensils are often involved.[2]


### 4.4 Nose picking

Nose picking is characterized by a **single fingertip (usually index) at the nose tip/nostril** with low velocity and micro-motion at that specific point.[1]

#### 4.4.1 Detection conditions

**Location:**

- Single-finger proximity: \(\tilde{\text{dist}}(indexTip, noseTip) < 0.16\).
- Other fingertips not clustered tightly (low cluster strength).

**Temporal pattern:**

- Dwell ≥ \(T_{nose}\) (e.g., 1–1.5 s) with low wrist velocity.
- Micro-motion amplitude of indexTip–nose distance \(A_{nose}\) in a modest band (e.g., \(0.01–0.03\) face widths), with a few direction changes.

**Wiping exclusion:**

- For nose wiping (short lateral swipes), the path of indexTip around the nose has larger lateral movement and higher peak velocity.
- Explicitly reject windows where either:
  - Peak indexTip velocity > a “fast swipe” threshold, or
  - The projected path length of indexTip around the nose over \(T_{nose}\) exceeds a maximum (e.g., \(> 0.1\) face widths), indicating a long swipe rather than a localized pick.


### 4.5 Lip picking / biting

Lip picking uses the index or thumb fingertip at the **lower lip**, with small repeated motions but usually not large mouth openings.[1]

#### 4.5.1 Detection conditions

**Location and shape:**

- \(\tilde{\text{dist}}(indexTip, lowerLip) < 0.16\) or \(\tilde{\text{dist}}(thumbTip, lowerLip) < 0.16\).
- Cluster strength low or moderate (few fingers involved).
- Pinch shape similar to nail biting but slightly lower required pinch strength, as lip picking may use a single finger.

**Temporal pattern:**

- Dwell ≥ \(T_L\) (1–2 s).
- Mouth opening small to moderate; avoid patterns with large repeated openings (eating veto).
- Micro-motion: fingertip–lip distance shows small oscillations, similar to nail biting.

Again, the **eatingLikely** negative detector is crucial to avoid false positives when drinking or biting food at the lips.


### 4.6 General face touching

General face-touch BFRBs are less clearly defined clinically and may overlap with normal behaviors.[1]
For a high-precision mode, it may be preferable to **only flag general face touching when it is both sustained and exhibits subtle micro-motions**, indicating an idle grooming or rubbing behavior rather than neutral resting.

#### 4.6.1 Detection conditions

- Hand near cheek, jaw, or temple zone for an extended dwell time (e.g., ≥ 3 s).
- Wrist velocity low, but fingertip centroid shows small micro-motions (amplitude \(0.01–0.03\) face widths, several direction changes).
- Exclude eatingLikely and restingLikely windows.

If a more conservative behavior is preferred, general face touching could be **tracked but not alerted**, reserving alerts for more clearly harmful BFRBs like hair pulling, skin picking, and nail/nose picking.


## 5. Strategies to Drive False Positives Toward Zero

### 5.1 Multi-stage scoring instead of single binary rules

Rather than firing a BFRB as soon as all conditions are barely met, assign a **confidence score** that aggregates:

- Proximity score (how close to key landmarks)
- Shape score (pinch vs cluster vs single-finger posture)
- Temporal score (dwell length, micro-motion pattern)
- Context score (penalty from eatingLikely or restingLikely)

Only trigger alerts when this score exceeds a high threshold (e.g., ≥ 0.9), and use hysteresis so that once triggered, it remains active until the score falls below a lower threshold (e.g., 0.6), reducing flicker.
This mirrors high-precision strategies used in BFRB wearables and eating detectors.[3][4][2]


### 5.2 Explicit “noise classes” and personalization

Gesture-recognition systems reduce false positives by training on many non-gesture motions and treating them as a **noise class**, then allowing per-user calibration.[6]
Even without a full ML model, a similar idea can be applied:

- Record short sessions of the user **eating, drinking, resting head on hand, adjusting glasses**, and treat these patterns as negative examples.
- Adjust thresholds (e.g., for micro-motion amplitudes, mouth-open ratios, and pinch strengths) so that these personal negative examples fall well below BFRB confidence.

This personalization is particularly effective because individuals differ widely in how they hold utensils, rest their face, or gesture.[6]


### 5.3 Trade detection delay for precision

Studies on just-in-time interventions emphasize that a small delay between the onset of a behavior and the alert is acceptable if it significantly reduces false positives.[4][2]
For this system, that means:

- Allowing 1–3 s windows to accumulate evidence (micro-motion patterns, chewing vs non-chewing) before firing, especially for nail biting, lip picking, and skin picking.
- Using the instantaneous pinch–pull rule only for hair pulling, where the motion is distinctive and the cost of a slight delay may be higher.


### 5.4 Use “hand-over-face” as a gating component

Work on hand-over-face detection shows that explicitly modeling when the hand occludes the face can be used to filter or veto other detectors.[7]
In this system, a simplified version can be implemented by checking if the bounding box of hand landmarks overlaps that of the face and if a significant fraction of face landmarks are spatially close to hand landmarks.

Only when “hand-over-face” is true should skin picking, lip picking, and general face-touching BFRBs even be considered, further reducing spurious triggers from hand motions near but not actually on the face.


## 6. Implementation Checklist

A practical path to integrating these ideas into the existing JavaScript engine is:

1. **Add mouth-opening metrics**: compute \(M_t\), its moving average and peaks.
2. **Implement micro-motion tracking** for grabPoint–zone distance, mouth opening, and thumb–index distance over sliding windows.
3. **Implement eatingLikely and restingLikely negative detectors** using the rules above.
4. **Refine hair-pulling logic** to enforce a clear net pull-back and filter out oscillatory scratching.
5. **Implement per-BFRB modules** (hair, skin, nail, nose, lip, general) that each output a confidence score.
6. **Combine scores** into a global BFRB score with hysteresis; only trigger when global score is high and negative detectors are off.
7. **Provide calibration UI** so users can tune sensitivity and optionally record personal negative examples (e.g., eating) to adjust thresholds.

Following these steps should significantly reduce false positives from eating and neutral face-resting while preserving sensitivity to true BFRB patterns, within the limits of what can be inferred from 2D hand and face landmarks alone.
